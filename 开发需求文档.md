AI量化交易平台开发文档


1.项目概述

本项目旨在构建一个基于 **Electron 桌面应用** 的面向加密货币的智能量化交易平台，面向散户投资者，提供从交易所账户绑定（Gate.io）、策略选择、自动交易、收益分析、交易记录的全流程方案。前端采用 Electron 实现跨平台桌面应用，后端采用 FastAPI + Gate.io 官方 Python SDK（gate-api）对接交易所。

项目目标	                         目标用户	         核心功能
为散户投资者提供简单易用的量化交易服务	个人投资者、散户	券商账户绑定、策略选择、自动交易、收益分析、交易记录

2.2 非功能需求
·  易用性：界面简洁直观，操作简单易懂
·  安全性：交易数据加密，账户信息安全
·  稳定性：交易系统稳定可靠，7×24 小时服务
·  响应速度：快速的数据更新和交易执行
·  图表性能：K 线图流畅渲染，支持数千根 K 线实时更新，目标 60fps
·  UI 响应：交互无卡顿，列表虚拟滚动，请求异步不阻塞

-关键业务流程说明：

交易所绑定流程：

用户发起绑定 → 选择模式（实盘/模拟）→ 输入 API Key + Secret → 授权交易权限 → 平台验证账户（调用对应 BaseURL）→ 绑定成功 → 同步账户数据 → 开始使用


交易模式切换流程：

用户点击切换 → 检查目标模式是否已绑定 → 已绑定则切换，未绑定则引导至账户设置绑定 → 切换后刷新账户总览、交易记录、持仓 → 实盘切换时弹出风险确认


策略订阅流程：

浏览策略列表 → 筛选策略 → 查看策略详情 → 了解风险收益 → 设置个性化参数（含执行模式：实盘/模拟）→ 确认订阅 → 策略开始运行 → 按所选模式通过对应 API 自动交易执行


交易执行流程：

策略信号产生 → 风控检查 → 按订阅的 mode 选择 broker（实盘/模拟）→ 生成交易指令 → 通过对应 Gate API 下单 → 订单状态监控 → 成交回报处理 → 更新持仓记录 → 生成交易流水


收益分析流程：

数据收集 → 收益计算 → 指标分析 → 可视化展示 → 对比分析 → 风险评估 → 生成报告 → 用户查看

6. Electron 与响应式设计

6.1 Electron 开发要点

· 主进程：窗口创建、菜单、系统托盘、IPC 桥接
· 预加载脚本：通过 contextBridge 安全暴露 API 给渲染进程
· 多窗口：欢迎页、登录页、主平台窗口（单例）
· 打包：electron-builder 生成 Windows/macOS/Linux 安装包

6.2 响应式设计考虑

· 使用 CSS Grid 和 Flexbox 布局
· 媒体查询适配不同屏幕尺寸（窗口可缩放）
· 触摸友好的交互设计（支持触屏设备）

| auth       | 否   | 420×520       | 登录/注册      |
| main       | 是   | 1280×800 最小 | 主平台（仪表盘、账户、策略、个人中心） |

· 主窗口关闭时退出应用；主窗口最小化可进托盘
· 使用 `BrowserWindow` 的 `webPreferences: { nodeIntegration: false, contextIsolation: true }`

6.3.3 主进程入口 (main.js) 职责

· 创建/销毁窗口、应用 ready/quit 处理
· 注册 IPC handlers（委托给 ipc-handlers.js）
· 系统托盘、全局快捷键（可选）
· 不执行重计算，仅做路由与转发


6.4 IPC 设计规范（高性能）

6.4.1 设计原则

· **小数据走 IPC**：< 64KB 的 JSON 可用 `ipcRenderer.invoke`，主进程用 `ipcMain.handle` 返回
· **大数据走 postMessage + Transferable**：K 线、行情等用 `webContents.postMessage` + `ArrayBuffer` 转移所有权，避免 Structured Clone 阻塞
· **渲染进程不阻塞**：耗时解析放入 Web Worker，主线程只做渲染

6.4.2 IPC 通道命名规范

| 通道前缀 | 方向         | 用途                     |
|----------|--------------|--------------------------|
| app:     | 渲染→主      | 应用级（窗口、设置）     |
| store:   | 双向         | 本地存储（偏好、Token）  |
| bridge:  | 主→渲染      | 主进程主动推送（postMessage） |

6.4.3 暴露给渲染进程的 API（preload.js）

```javascript
// 仅暴露必要方法，避免大对象传递
contextBridge.exposeInMainWorld('electronAPI', {
  // 窗口
  app: {
    minimize: () => ipcRenderer.invoke('app:minimize'),
    maximize: () => ipcRenderer.invoke('app:maximize'),
    close: () => ipcRenderer.invoke('app:close'),
    openMain: (token) => ipcRenderer.invoke('app:openMain', token),
  },
  // 本地存储（小数据）
  store: {
    get: (key) => ipcRenderer.invoke('store:get', key),
    set: (key, val) => ipcRenderer.invoke('store:set', key, val),
    // 常用 key：token, trading_mode(real|simulated)
  },
  // 大数据接收：主进程 postMessage 推送，渲染进程监听
  onChartData: (callback) => { /* 见 6.4.5 */ },
});
```

6.4.4 大数据传输（K 线/行情）流程

```
后端 WebSocket → 主进程 Node → ArrayBuffer 解析
    → webContents.postMessage(data, [buffer])
    → 渲染进程 onmessage 接收
    → Web Worker 解析/聚合
    → 主线程 lightweight-charts.update()
```

· K 线增量：主进程只转发 ArrayBuffer，不 JSON 解析
· 历史 K 线：REST 返回后，主进程可做流式转发，或渲染进程直接 fetch

6.4.5 IPC 消息清单

| 通道              | 参数                    | 返回值/行为              |
|-------------------|-------------------------|--------------------------|
| app:minimize      | -                       | -                        |
| app:maximize      | -                       | -                        |
| app:close         | -                       | -                        |
| app:openMain      | token: string           | -                        |
| store:get         | key: string             | any                      |
| store:set         | key, value              | -                        |
| store:trading_mode| -                       | 读取/写入 trading_mode   |
| bridge:chartData  | (主→渲染) symbol, buffer| 无，渲染进程 onmessage   |


6.5 图表与 UI 性能优化规范

6.5.1 K 线图选型：lightweight-charts

· 库：`lightweight-charts`（TradingView 官方，npm install lightweight-charts）
· 特点：约 35KB、Canvas 渲染、支持数千条 K 线实时更新
· 数据格式：`{ time: '2024-01-15' 或 1705305600, open, high, low, close }`

6.5.2 图表性能规范

| 场景           | 策略                                         |
|----------------|----------------------------------------------|
| 初始加载       | 最多加载 500 根 K 线，按需分页拉取更早数据   |
| 实时推送       | 仅 append/update 最新一根，不用 setData 全量替换 |
| 缩放/平移      | 使用图表内置交互，避免自定义拖拽阻塞         |
| 多图同屏       | 单页不超过 3 个图表实例，其余用 Tab 切换     |
| 数据解析       | 在 Web Worker 中解析，主线程只调用 update()   |

6.5.3 UI 响应规范

· 所有网络请求：异步，带 loading 状态，不阻塞 UI
· 列表虚拟滚动：自选币、交易记录等长列表使用虚拟滚动（如 100 条可见）
· 防抖/节流：搜索、筛选输入 300ms 防抖；行情刷新 100ms 节流
· 动画：使用 CSS transform/opacity，避免触发布局重排

· 策略订阅时指定 mode，该策略仅在该模式下执行
· 当前交易模式（current_mode）决定账户总览、交易记录、持仓的展示来源

12. 框架与规范汇总

12.1 前端（Electron 渲染进程）

· 图表：lightweight-charts，K 线用 addCandlestickSeries，增量用 update
· 大数据：Web Worker 解析，主线程仅调用 chart.update()
· 请求：统一走 api.js 封装，支持 Token、错误处理
· 本地状态：小数据用 electronAPI.store，大数据不存主进程

12.2 主进程与 IPC

· 小数据：ipcMain.handle + ipcRenderer.invoke
· 大数据：webContents.postMessage + ArrayBuffer 转移
· 仅暴露 app、store、onChartData 等必要 API

12.3 后端 API

· 基础路径 /api/v1，认证 Header: `Authorization: Bearer <token>`
· 交易所绑定支持 mode=real|simulated，对应 Gate.io 实盘/Testnet
· 交易模式：PUT /broker/mode 切换；portfolio、assets、trading 等接口支持 mode 参数，不传则用用户当前模式
· 策略订阅需传 mode，策略执行时按订阅的 mode 选择 broker
· K 线接口 /market/candlesticks 必须支持 limit、from、to，响应数组格式
· 交易流水 status：open（未成交）、closed（已成交）、cancelled（已撤单）
· WebSocket 支持 ticker、order、balance、nav、kline、positions 事件
