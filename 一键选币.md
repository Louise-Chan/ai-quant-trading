# 一键选币 - 开发需求文档

## 1. 功能概述

**一键选币** 通过规则引擎、机器学习模型、AI Agent 三层筛选机制，从全市场加密货币中自动选取少量优质标的，帮助用户快速构建高质量自选池。

| 目标 | 说明 |
|------|------|
| 输入 | Gate.io 全市场交易对（数百至上千个） |
| 输出 | 3～15 个优质币种，一键加入自选 |
| 筛选维度 | 流动性、波动率、趋势强度、资金流向、技术指标、AI 综合评分 |

## 3. 技术架构

### 3.1 三层筛选机制

```
                    ┌─────────────────────────────────────┐
                    │           用户点击「选币」           │
                    └─────────────────────────────────────┘
                                        │
                                        ▼
┌───────────────────────────────────────────────────────────────────────────┐
│ 第一层：规则引擎 (Rule Engine)                                              │
│ - 流动性过滤（24h 成交额 > 阈值）                                            │
│ - 波动率过滤（排除异常波动）                                                  │
│ - 交易对过滤（仅 USDT 交易对）                                               │
│ - 输出：候选池 50～200 个                                                    │
└───────────────────────────────────────────────────────────────────────────┘
                                        │
                                        ▼
┌───────────────────────────────────────────────────────────────────────────┐
│ 第二层：机器学习模型 (ML Model)                                             │
│ - 特征：技术指标、资金流向、历史表现、波动率等                               │
│ - 模型：分类/回归模型（如 XGBoost、LightGBM）或预训练嵌入                   │
│ - 输出：每个候选的得分排序，取 Top N                                         │
└───────────────────────────────────────────────────────────────────────────┘
                                        │
                                        ▼
┌───────────────────────────────────────────────────────────────────────────┐
│ 第三层：AI Agent（可选，用户点击「接入Agent」时启用）                        │
│ - 基于 LLM 的综合分析                                                       │
│ - 结合用户偏好、市场情绪、宏观信息                                           │
│ - 输出：最终推荐列表 + 自然语言解释                                          │
└───────────────────────────────────────────────────────────────────────────┘
                                        │
                                        ▼
                    ┌─────────────────────────────────────┐
                    │  返回 3～15 个币种，前端展示并确认   │
                    └─────────────────────────────────────┘
```

## 4. 规则引擎设计

### 4.1 规则列表

| 规则 ID | 规则名称 | 条件 | 说明 |
|---------|----------|------|------|
| R1 | 交易对过滤 | `quote == "USDT"` | 仅 USDT 交易对 |
| R2 | 流动性过滤 | `24h_volume_usdt >= 1_000_000` | 24h 成交额 ≥ 100 万 USDT |
| R3 | 波动率过滤 | `0.02 <= volatility_24h <= 0.5` | 排除极端波动 |
| R4 | 价格过滤 | `price >= 0.00001` | 排除极低价币 |
| R5 | 涨跌幅过滤 | `-0.3 <= change_24h <= 0.3` | 可选：排除单日暴涨暴跌 |

### 4.2 实现要点

- 规则可配置（数据库或配置文件）
- 支持 AND/OR 组合
- 输出：通过规则的 symbol 列表

### 4.3 伪代码

```python
def apply_rules(tickers: list, rules: list) -> list:
    candidates = []
    for t in tickers:
        if all(rule.evaluate(t) for rule in rules):
            candidates.append(t)
    return candidates
```

## 5. 机器学习模型设计

### 5.1 特征工程

| 特征类别 | 特征示例 |
|----------|----------|
| 价格相关 | 当前价、24h 涨跌幅、7d 涨跌幅 |
| 成交量 | 24h 成交量、成交量变化率 |
| 波动率 | 24h 波动率、历史波动率 |
| 技术指标 | RSI、MACD、布林带位置 |
| 资金流向 | 买卖量比、大单占比（如有） |

### 5.2 模型选型

- **方案 A**：轻量级 XGBoost/LightGBM 分类模型，预测「是否优质」
- **方案 B**：回归模型输出得分，按得分排序
- **方案 C**：无监督聚类（如 K-Means）后，选取代表性样本

### 5.3 训练与更新

- 训练数据：历史 ticker + K 线 + 人工标注或历史表现
- 更新频率：可每日/每周离线训练，模型文件存于 `backend/models/`
- 推理：实时加载模型，对候选池打分

### 5.4 输出格式

```json
{
  "symbol": "BTC_USDT",
  "score": 0.85,
  "reason": "高流动性、趋势向上、波动适中"
}
```

## 6. AI Agent 设计

### 6.1 职责

- 结合规则引擎 + ML 结果，做最终推荐
- 支持用户偏好输入（稳健/激进、偏好板块等）
- 生成自然语言解释，增强可解释性

### 6.2 调用方式

- **方案 A**：接入 OpenAI API / 兼容接口（如 Azure OpenAI、本地模型）
- **方案 B**：接入 LangChain / LlamaIndex 等框架，支持多模型切换

### 6.3 Prompt 设计

```
你是一个加密货币选币助手。根据以下候选池（已通过规则和 ML 筛选）和用户偏好，推荐 3～10 个优质币种。

候选池：[{symbol, score, reason}, ...]
用户偏好：{user_preference}

请输出：
1. 推荐列表（symbol + 简要理由）
2. 风险提示
```

### 6.4 输出格式

```json
{
  "recommendations": [
    {"symbol": "BTC_USDT", "reason": "流动性最佳，趋势稳健"},
    {"symbol": "ETH_USDT", "reason": "生态活跃，近期表现强"}
  ],
  "summary": "基于当前市场，推荐以上币种，请注意分散风险。"
}
```

## 12. 前后端对接说明

### 12.1 接口清单

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/dashboard/smart-select` | POST | 一键选币（规则引擎） |
| `/api/v1/dashboard/agent-select` | POST | Agent 选币 |
| `/api/v1/dashboard/watchlist/batch` | POST | 批量加入自选 |

### 12.2 请求头

所有接口需携带登录态：

```
Authorization: Bearer <token>
Content-Type: application/json
```

### 12.3 接口详情

#### smart-select

**请求**：
```json
{
  "top_n": 10,
  "mode": "real"
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| top_n | int | 否 | 推荐数量，默认 10 |
| mode | string | 否 | real/simulated，默认取用户当前交易模式 |

**响应**：
```json
{
  "success": true,
  "data": {
    "symbols": [
      {"symbol": "BTC_USDT", "reason": "高流动性、趋势向上", "score": 0.85}
    ],
    "source": "rule_engine"
  },
  "message": "ok",
  "code": 200
}
```

### 12.4 错误码

| code | 说明 |
|------|------|
| 200 | 成功 |
| 400 | 参数错误 |
| 401 | 未登录 |
| 500 | 服务器错误 |

### 12.5 前端对接流程

```
用户点击「选币」
    ↓
获取当前交易模式 (broker.status)
    ↓
POST /dashboard/smart-select { top_n: 10, mode }
    ↓
展示弹窗（symbols + reason）
    ↓
用户勾选 → 点击「加入自选」
    ↓
POST /dashboard/watchlist/batch { symbols: [...] }
    ↓
刷新自选列表
```

### 12.6 后端服务调用链

```
dashboard.smart_select
    ↓
get_mode(db, uid)  → 获取用户当前模式
    ↓
list_tickers(mode)  → Gate.io 行情
    ↓
apply_rules(tickers)  → 规则引擎筛选
    ↓
返回 Top N
```
